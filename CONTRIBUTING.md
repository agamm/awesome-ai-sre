# Contribution Guidelines

Thank you for contributing to Awesome AI SRE!

## Adding a new entry

- Search existing entries to avoid duplicates.
- Add the entry to the appropriate category in alphabetical order.
- Use the format: `| [Name](https://example.com) | One-line description. |`
- The description should be concise — one sentence, ending with a period.
- The tool or resource must be relevant to AI-powered site reliability engineering.

## Quality standards

- The tool, service, or resource should be actively maintained.
- Commercial products should have a public website with clear documentation.
- Open source projects should have at least 50 stars or be part of a recognized foundation (CNCF, Linux Foundation, etc.).
- Blog posts and articles should provide lasting value, not just news.

## Adding a new category

- If your entry doesn't fit an existing category, propose a new one in your PR.
- Categories should have at least 3 entries to justify their existence.

## Pull request process

1. Fork the repository.
2. Add your entry following the guidelines above.
3. Submit a pull request with a clear title (e.g., "Add ToolName to AI SRE Agents").
4. One entry per PR is preferred but not required.

## Removing entries

If a tool has been discontinued, acquired and shut down, or is no longer maintained, please submit a PR to remove it. Include a brief note in the PR description about why the entry should be removed.
